## Commands

Run from repo root:

- `pnpm wgsl-check` — validate all `.wgsl` files via `naga --bulk-validate` (`scripts/wgsl-check.mjs`). Runs before build + `dev:ui`.
- `pnpm build:native` — `zig build -Doptimize=ReleaseSafe` in `native/`, copy `libriptide.so` → `dist/native/riptide.node` (the napi addon). Also copies the bundled `mock.vcd` (+ its sidecar) into `dist/native`. (The `query-fixture` test exe is no longer copied here — the differential test reads it from `native/zig-out/bin/`.)
- `pnpm build:native:win` — cross-compile the addon for Windows (`-Dtarget=x86_64-windows`) via the napi shim (`scripts/gen-win-napi-shim.mjs`) → `dist/native/riptide-win.node` (separate name; both binaries bundle, only the host-matching one is `require`d).
- `pnpm build:native:mac` — cross-compile the addon for macOS (`-Dtarget=aarch64-macos`) → `dist/native/riptide.node` (same name as Linux; `build.zig` allows undefined napi symbols on Mach-O via `linker_allow_shlib_undefined`). Untested — packaging the dmg still requires running electron-builder on macOS.
- `pnpm build` — wgsl-check + build:native + compile main process (`tsc`, CommonJS → `dist/main`) + bundle renderer (`scripts/build-ui.mjs`, esbuild + Tailwind) + copy `index.html`.
- `pnpm dev` — build + launch Electron. `pnpm dev:blank` — same but `RIPTIDE_NO_TRACE=1` (boots the idle/no-file UI even though the mock is present).
- `pnpm dev:mac` — run-from-source path for **macOS** (the default `build:native` copies a Linux `.so`, so it can't run on a Mac): host-arch `zig build` → `libriptide.dylib` → `riptide.node`, then main + renderer (skips `wgsl-check`, so no `naga`/Rust needed) + launch. Needs `zig` + the `../tide` / `../tide-vcd` siblings, like every native build.
- `pnpm dev:ui` — wgsl-check + build:native + renderer-only esbuild watch & live-reload at `http://localhost:5173` (`scripts/dev-ui.mjs`). Pair with Electron started under `RIPTIDE_DEV=1` to loadURL the dev server instead of `dist/renderer/index.html`.
- `pnpm typecheck` — two tsconfigs (`tsconfig.json` main, `tsconfig.renderer.json` the renderer = SolidJS app + shared `.ts`), no emit.
- `pnpm test` / `pnpm test:visual` / `pnpm canvas-test` — see `TESTING.md` (oracle suites; DOM screenshot regression; GPU golden).
- `pnpm dist` / `pnpm dist:dir` — build:prod + electron-builder → `dist/installers/` (AppImage on Linux). `dist:linux` / `dist:win` / `dist:mac` / `dist:all` target specific platforms (each runs the matching `build:native:*` first; `dist:all` = linux + win). `dist/native/**` is asar-unpacked. **The `mac` (dmg) target's native addon now cross-builds (`build:native:mac`), but electron-builder can only assemble the dmg when run on macOS — so `dist:mac` is not fully buildable from Linux.**

Requires `naga` CLI (`cargo install naga-cli`) and `zig` on PATH (build-time only — testers running a packaged build need neither). The native build pulls `tide` / `tide-vcd` as Zig deps — sibling repos at `../tide`, `../tide-vcd` (referenced from `native/build.zig.zon`). No linter; tests via `pnpm test` (oracle + visual harnesses, see `TESTING.md`).

## Architecture

Single-package repo at root (`riptide`). Electron app rendering digital waveforms via WebGPU. Waveform data comes from a real VCD on disk, parsed by tide-vcd in a Zig napi addon.

**Main process** — `src/main/index.ts`. Enables `enable-unsafe-webgpu` (+ Vulkan on Linux), opens BrowserWindow. The trace path is carried to the renderer in the window URL (`?vcd=<abs path>`); defaults to the bundled `native/src/mock.vcd`. Loads `http://localhost:5173/?vcd=…` when `RIPTIDE_DEV` set, else `dist/renderer/index.html`. The `riptide:open-vcd` IPC handler shows a native file dialog and returns the chosen path — the renderer then swaps the trace **in place** (no window reload; see `swapTrace` below).

**Native addon** (`native/src/`, Zig → `dist/native/riptide.node`): `main.zig` registers the napi surface — `loadVcd(path)`, `getMockSegments(specs)`, `getHierarchy()`, `getValueAt(handle, tick)`, `getEdges(handle, start, count)` (up to `count` transitions at/after `start`, via tide's `db.queryNext` — a cheap prefix read for clock period/phase + reset-band detection; see `wave/clock.ts`). `mock_db.zig` parses a VCD path via tide-vcd into a `tide.Database` + mirrored `hier.zig` hierarchy (cached; `loadVcd` swaps it, leaving the prior trace intact on a bad open). `pack.zig` `packSignal()` turns a signal's tide transitions into a row-agnostic `seg.PackedSignal` (segments with row bits 0, value samples, labels); `segments.zig` owns the pooling/finalize (see Conventions). **Pack cache:** `getMockSegments` keys each `PackedSignal` by `PackKey` (handle + kind/shade/gate + label radix/enums-hash — everything *but* the row) in a module-global `pack_cache`, so an add/remove/reorder/radix-change repacks (tide query + flag compute + gate queries + label format) **only the changed signal**; the rest are reassembled from cache via `Scene.pushPackedSignal` (OR the row into `row_flags`, append samples/labels — no tide query). Cleared on `loadVcd` (new trace invalidates handles/values). `label.zig` `formatValue` formats each multi-bit value label natively (byte-for-byte port of the renderer's `formatSegmentValue` — keep the two in sync), called from `packSignal` in lockstep with the segments; `getMockSegments` returns them as `labelBytes` (concatenated ASCII) + `labelOffsets` (`multiCount+1` prefix offsets, label *i* = `labelBytes[off[i]..off[i+1]]`), so the renderer does **no** per-segment `getValueAt`/JS formatting for pill labels. The pack spec carries the per-row `radix` + `enums` table the formatter needs. V8's Electron sandbox rejects external pointers, so buffers are `napi_create_arraybuffer` (V8-owned) + memcpy.

**Renderer** — `src/renderer/index.tsx` → `App.tsx`. **SolidJS** (JSX lowered by `babel-preset-solid` via the esbuild plugin `scripts/esbuild-solid.mjs`). App state is a **Zustand** store (`store/store.ts`) exposed to components through `solid-zustand` (`useAppStore()`, fine-grained reactive reads) and read synchronously by the rAF loop / native handlers via `useAppStore.getState()` / `.setState()` — no stale-closure problem, so there are no state/ref mirrors. The persisted document slice maps 1:1 onto the sidecar (`selectSidecarText` + `startAutosave`). Renderer styling is **Tailwind v4**, compiled by the esbuild plugin `scripts/esbuild-tailwind.mjs` from `src/renderer/index.css` → `dist/renderer/index.css` (linked from `index.html`); utilities use a `tw:` prefix and preflight is disabled (`@apply` hybrid for shared component rules). The **framework-agnostic** data/GPU layers (`gpu/`, `hier/`, `native.ts`, `perf.ts`, `runtime.ts`) are plain `.ts` sitting alongside the Solid components, all under `src/renderer/`:

- `runtime.ts` — reads `VCD_PATH` / `SIDECAR_PATH` from the window URL once at module load to bootstrap the first trace.
- `native.ts` — typed bridge to the addon; marshals `getHierarchy` raw nodes into the `Hierarchy` map, exposes `loadVcd`, `getMockSegments`, `getValueAt`.
- `hier/` — trace/view model. `types.ts` (`Hierarchy`, `Scope`, `Signal`, …), `hierarchy.ts` (lookups), `scene.ts` (builds `SCENE` = hierarchy + active signal rows + pack specs; `INITIAL` = cursor/markers/time window), `sidecar.ts` (persistence). `SCENE`/`INITIAL`/`SIDECAR` are `let` exports — `swapTrace(path)` reassigns them (reload native db → reload sidecar → rebuild scene). On an in-app "Open VCD…", `App.handleOpenVcd` calls `swapTrace` then `store.resetForTrace()` (one atomic `setState` re-seeding the document slice from the new `SCENE`/`INITIAL` + bumping `traceNonce`); the canvas's `traceNonce` subscription resets the viewport, and the `activeSignals` subscriptions repack — no window reload.
- Components (all under `src/renderer/`): `App.tsx` (layout grid, panel collapse/resize, Open-VCD swap), `wave/WaveCanvas.tsx` (GPU init + the rAF render loop), `wave/viewport.ts` (pan/zoom/fit controller, transient render state outside the store), `ActiveSignals.tsx` + `ActiveSignal.tsx` (signal rows; per-row value cell is a `createMemo` on `cursorTicks`), `SignalTree.tsx` (virtualized + slide-animated hierarchy tree, keyed by node id), `WavesToolbar.tsx`, `MarkersBar.tsx`, `EditableNum.tsx`, `ColorPicker.tsx`, `ClockPicker.tsx` (clock-format/timebase picker), `EnumDialog.tsx` (per-row enum-table editor), `ContextMenu.tsx`, `MenuBar.tsx`, `GlobalTooltip.tsx`, `HoverReadout.tsx`, `PerfOverlay.tsx`. Value/enum-label formatting in `wave/value.ts`, ruler/clock/time helpers in `wave/format.ts`, packed colors in `wave/palette.ts`, layout/zoom constants in `wave/constants.ts`. Canvas PNG snapshot (Save Canvas / Export) via `wave/capture.ts` (drained by the rAF loop right after `renderFrame`); the `window.__bench` pack-cost harness lives in `bench.ts`.

**WebGPU pipeline** (`src/renderer/gpu/`):

- `device.ts` — `initGPU(canvas)` sets up adapter/device/ctx (requests `timestamp-query` when available); `resizeCanvas` syncs the backing store with CSS size × DPR.
- `data.ts` — `Viewport` shape + viewport packing (`writeViewportInto`, `VIEWPORT_BYTES = 48`). The timeline end is the trace's last event time (native `end_t` → `SCENE.hierarchy.endTicks` → `TRACE_END`); the clock period/phase is the detected timebase grid (`store.clockGrid`, from `wave/clock.ts` `detectClockGrid` via native `getEdges`) — neither is a hardcoded constant.
- Native packing (`native/src/segments.zig`) — produces GPU-ready `PackedSegment` (3×u32: `t_start`, `t_end`, `row_flags`), `RowInfo` (5×u32 per row), and shared byte-stride sample pools `x0Pool` / `x1Pool` (LSB / MSB). Per-row `bytes_per_sample = ceil(bit_width / 8)` (= tide's `Type.bytes()`): each sample is that many consecutive bytes — tide's native per-sample byte run, **memcpy'd straight in** (`pack.zig setSamples` copies a signal's whole `query.x0s`/`x1s` plane in one shot; no per-sample repack), so signals of any width are carried in full. The pools are bound as `array<u32>` (WGSL can't bind byte storage); the shader byte-addresses them, decoding `(lsb, msb)` per instance via `RowInfo.x0_offset` + `segment_start` + `bytes_per_sample`, OR-folding the sample's bytes (it only needs whole-sample non-zeroness to pick line/crosshatch/color). Each pool is padded to a 4-byte multiple (`finalize`) so `writeBuffer` accepts it. The CPU value path (`getValueAt`/`jsWordArray`) is independent and still returns u32 word arrays.
- `digital.ts` + `digital.wgsl` — one shader module; a `VARIANT` override constant (0 single, 1 multi) selects the variant at pipeline-build time. Vertex entry `vs_main`; fragment entries `fs_single` (1-bit waveform line) / `fs_multi` (multi-bit pill w/ rounded SDF). Triangle-strip rects, 4 verts × N instances. `buildPipelineFromPacked` compiles a variant async with its own segment storage buffer; `rebindPipeline` reuses an already-compiled pipeline with a fresh segment buffer (synchronous — so an active-signal add/remove repacks on the spot). Both variants share the per-scene `RowInfo` + sample pools from `createSceneBuffers`. 6 bind-group entries (uniform + 5 read-only-storage); alpha blending on. `wave/WaveCanvas.tsx` feeds `getMockSegments(buildPackSpecs())` from the addon — 1-bit rows go to the single buffer, wider rows to the multi buffer.
- `lines.ts`/`lines.wgsl`, `rect.ts`/`rect.wgsl`, `text.ts`/`text.wgsl` — the other instanced batch renderers (grid/cursor/marker lines, notch/pill rects, glyph atlas text). `colors.ts` — per-row color storage buffer (`MAX_ROWS`). `timing.ts` — `GpuTimer` (WebGPU `timestamp-query`, n/a fallback).
- `frame.ts` — `renderFrame` writes the viewport uniform once, then one render pass with clear: `linesBg`, `rectsBg`, each digital pipeline, `textBody`, `linesFg`, then per-pill `rect`+`text` overlays (each pill's own pair so its rect occludes earlier pills — painter's algorithm, no z buffer). Optional `GpuTimer`.

`wave/WaveCanvas.tsx` runs `requestAnimationFrame` (in `onMount`, after GPU init). Canvas size (`clientWidth`/`clientHeight`) is read live each frame; row height is `ROW_HEIGHT_CSS` (`wave/constants.ts`, mirrors the `--row-h` CSS var so canvas rows line up with the DOM `.s-row`s). `resizeCanvas` syncs the backing store on `ResizeObserver`; DPR-only changes (drag between displays) tracked via `matchMedia('(resolution: …dppx)')`, re-armed each fire. The active-signal value column is computed CPU-side via `valueAtTick` (native `getValueAt`) + `formatSegmentValue` (`wave/value.ts`, same LSB/MSB word convention as the shader).

**Perf overlay** (`perf.ts`, `gpu/timing.ts`, `PerfOverlay.tsx`): `perf.ts` lives at `src/renderer/` (not `gpu/`). Toggle the HUD with the backtick **`` ` ``** key, `?perf=1` in the URL, or `window.__perf.enable()`; `window.__perf.dump()`/`.snapshot()`/`.reset()` from DevTools. Tracks electron (main-thread rAF) present fps + dropped frames + longtask jank, canvas GPU pass ms (via `frame.ts`'s optional `GpuTimer`; n/a when unsupported), CPU encode ms, the trace-load phase breakdown (`stamp()`s finalized on first frame), VCD-swap phases (`swapMark`s in `swapTrace`), and add-signal latency (click → repack → present). Enable state persists in `sessionStorage`; frame metering + load stamps are always-on (cheap), GPU queries + overlay only when enabled.

## Conventions

- **Split tsconfigs**: `tsconfig.json` main = CommonJS to `dist`; `tsconfig.renderer.json` = the renderer (SolidJS app + shared `.ts`), `jsx:preserve` + `jsxImportSource:solid-js`. ESNext + Bundler resolution, no emit.
- **WGSL as text import**: esbuild `--loader:.wgsl=text`; types in `gpu/wgsl.d.ts`.
- **CPU/GPU layout must stay in sync** across Zig + WGSL: `PackedSegment` = 3×u32 (`t_start`, `t_end`, `row_flags`); `RowInfo` = 5×u32 (`x0_offset`, `x1_offset`, `bytes_per_sample`, `segment_start`, `flags`) — offsets are BYTE offsets into the (u32-typed) pools, `bytes_per_sample = ceil(bit_width/8)`. The same struct is also declared (read-only, `flags` only) in `labels.wgsl`, so keep all three (`segments.zig`, `digital.wgsl`, `labels.wgsl`) in sync. `RowInfo.flags` bit 0 = row dim (eye toggle); native emits 0 and the renderer sets it directly via `setDimFlags` (a tiny `writeBuffer`, no repack — so it scales past 32 rows, unlike the old viewport bitmask). Viewport = 12 × 4 B = 48 B (16-aligned): `ticks_per_pixel:f32`, `start_ticks_int:i32`, `start_ticks_frac:f32`, `width:f32`, `height:f32`, `row_height:f32`, `dpr:f32`, `selected_row:i32`, `wave_y_offset:f32`, then 3×f32 pad (slots 9–11; row dimming moved to `RowInfo.flags`). `start_ticks` is split int/frac so shader subtraction (`f32(i32(t_start) - start_ticks_int) - start_ticks_frac`) keeps full integer precision for tick values > 2^24. Touch one side = update the other (`native/src/segments.zig`, `digital.wgsl`, `writeViewportInto` in `gpu/data.ts`).
- **Segment rowFlags packing**: `[15:0]` row index, `[16]` shade, `[17]` right edge, `[18]` rising edge, `[19]` falling edge, `[20]` mute. Logic values use an LSB/MSB pair per bit: `(m,l)` = `(0,0)` 0, `(0,1)` 1, `(1,0)` x, `(1,1)` z. Shader bits 16..23 of `row_flags` repack into `VertexData.flags[0..7]`; higher `VertexData.flags` bits come from per-sample decoded LSB/MSB and differ between the single and multi variants.
- **CSS-pixel + DPR contract**: all viewport dims (`width`, `height`, `row_height`, `ticks_per_pixel`) and all vertex/geometry coords fed to shaders are CSS px. Shaders divide by `viewport.width`/`height` (CSS) to get clip space; since the framebuffer is `clientSize × dpr`, the clip→framebuffer step *already* scales by DPR. Therefore **shader size literals (line/border thickness, gaps, corner radius, hatch spacing) are bare CSS px and must NOT be multiplied by `viewport.dpr`** — doing so double-applies DPR (a "1px" stroke becomes `dpr²` device px) and breaks cross-display consistency. The `dpr` uniform is retained for layout but currently unused by shaders. DPR is applied in exactly one place: `resizeCanvas` (`device.ts`) sizing the backing store. Text crispness is handled by rendering the glyph atlas at `2×dpr` resolution (`text.ts`), not by scaling cell geometry.
- **Vertical-line alignment contract**: the ruler is the source of truth for where a line lands. `xForTick(t) = (t - startTicks) / ticksPerPixel` maps a logical time instant to its CSS-px x. `LINE_THICKNESS_CSS` (`wave/constants.ts`, = `thickness` in `lines.wgsl` — keep in sync) is how far a line extends.
  - **Time-aligned lines** (ruler notches, dashed grid, cursor, markers) are **left-aligned** to their logical time: the line's *left edge* sits on `xForTick(t)` and it extends `THICKNESS` px to the right. `lines.wgsl` left-aligns (`x_px + corner_x * thickness`), so CPU code passes `l.x = xForTick(t)` directly (no half-thickness inset). Notch rects use the same `r.x = xForTick(t)`, `w = LINE_THICKNESS_CSS`.
  - **Segment edges** (multi-bit gap; single-bit right edge / rising-falling caret) are **right-justified** to the segment, drawn *leading up to* the next segment's boundary `xForTick(t_end)`. The segment body is left-aligned to `xForTick(t_start)`, so each segment is itself left-aligned to its logical time. Segments are NOT centered (gaps/edges never straddle the boundary). This is already how `digital.wgsl` packs `pixel_bounds` (`-xgap_px` on the right; `edge_left_x = half - thickness`; caret apex biased `- thickness*0.5`) — no change needed there.
  - Consequence: segment edges deliberately do **not** line up with grid lines — a grid line's left edge starts on the boundary going right, while the segment edge ends on the same boundary coming from the left.
  - **Hover guide** is the one exception: it is **centered** on the pointer. It's still drawn with the left-align shader, but `updateHover` biases its tick left by `LINE_HALF_CSS` px so the left-aligned line renders centered on the pointer pixel. The reported logical time is thus ~½-thickness off the raw pointer pixel. `tickAtClientX` applies the **same** bias, so click-to-place cursor/markers land exactly where the centered hover line sat. Pill anchoring (`addFlag`), marker grab-test center, and span-arrow endpoints all account for the left-edge convention (add `LINE_HALF_CSS` to reach the visual center).
- **Trace vs. sidecar**: hierarchy + waveform samples + enum integer values are *trace* data (the VCD, via tide). The **sidecar** (`<trace>.sidecar.json`, next to the trace) carries only the presentation overlay — which signals are active, their order/color/radix/role, plus cursor/markers/time window and UI chrome. Signals are keyed by hierarchical **path** (not run-specific handles or row indices), so a sidecar opens against any run of the same design whose paths still resolve. Opening the bundled mock loads its curated view; opening a fresh trace finds no sidecar and starts empty. `scene.ts` `ROWS`/`ENUM_TYPES` hold TS-only metadata (enum int→label tables, gating, a curated default view) that tide doesn't yet carry — temporary shims, see `TIDE_INTEGRATION.md`. Sidecar is auto-written on change (`writeSidecarFile`).
- **Time unit**: ticks = nanoseconds (integer). The bundled mock's clock has a 10 ns period / first edge at tick 5 (detected, not hardcoded) and runs to tick 90 (its last event time, surfaced as `endTicks`).
